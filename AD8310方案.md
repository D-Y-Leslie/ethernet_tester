## 6. AD8310 模块识别与端口定义

模块：

```text
AD8310 Figure 38 DC-Coupled Log Amp v2
```

参考 PCB 丝印：

```text
Ji       = 左 SMA
Jo       = 右 SMA
TPin     = 输入测试点
TPVout   = 输出电压测试点
TPcom1/2 = GND
JDCV+    = 电源正端
Btrim    = offset / baseline 调节相关测点或旋钮节点
Ro       = 输出相关节点
Co       = 输出滤波相关节点
```

已确认端口关系：

```text
右 SMA 中心针 -> TPVout = 蜂鸣
左 SMA 中心针 -> TPin = 2.52k ohm
右 SMA 中心针 -> TPin = 11.1k ohm
左 SMA 中心针 -> TPVout = 8.1k ohm
```

重要结论：

- 右 SMA / Jo 与 TPVout 直接相关，是输出端。
- 右 SMA 不能作为 RF 输入。
- 后续不要再把信号源接到右 SMA。
- 左 SMA / Ji 是输入侧候选端，与 TPin 有电阻关系。
- 但左 SMA 是否就是当前模块的有效 RF 输入端，仍需要进一步确认。

## 9. 左 SMA 与右 SMA 注入测试结果

左 SMA 注入 GPIO40：

```text
GPIO40 -> 串联电阻 -> 左 SMA 中心针

OFF:
左 SMA 中心针 对 TPcom = 0 V，可能有 1.x mV
TPVout 对 TPcom = 2.18 V

ON:
左 SMA 中心针 对 TPcom = 0 V，可能有 1.x mV
TPVout 对 TPcom = 2.17 V
```

补充测量：

```text
ON 时：
GPIO40 引脚 对 TPcom = 1.58 V
串联电阻靠 GPIO40 端 对 TPcom = 1.58 V
串联电阻靠左 SMA 端 对 TPcom = 0 V
串联电阻靠左 SMA 端 对左 SMA 中心针 = 蜂鸣
```

右 SMA 注入 GPIO40 时出现异常：

```text
OFF:
右 SMA 中心针 对 TPcom = 0 V
TPVout 对 TPcom = 0 V

ON:
右 SMA 中心针 对 TPcom = 0 V
TPVout 对 TPcom = 0 V
```

随后确认：

```text
右 SMA 中心针 -> TPVout = 蜂鸣
```

恢复验证：

```text
把右 SMA 中心针换到左后，TPVout 恢复到 2.18 V
```

可靠结论：

- 右 SMA 注入等于把 GPIO40 接到了输出端 TPVout，会把输出拉低。
- 右 SMA 不能作为输入端。
- 左 SMA 是输入候选端，但用低频 GPIO 注入不能判断 RF 检波是否有效。

## 10. Pluto RF 输入 AD8310 的测试结果

### 10.1 Pluto 经左 SMA 输入

接法：

```text
Pluto TX1A -> 同轴线 -> AD8310 左 SMA
右 SMA 不接
万用表测 TPVout 对 TPcom
```

结果：

```text
TX OFF:       TPVout = 2.73 V
TX ON -40dB:  TPVout = 2.73 V
TX OFF:       TPVout = 2.73 V
TX ON -20dB:  TPVout = 2.73 V
TX OFF:       TPVout = 2.72~2.73 V
TX ON -10dB:  TPVout = 2.72~2.73 V
```

结论：

- Pluto 经左 SMA 输入时，TPVout 没有可观测变化。
- 不能证明左 SMA 当前接法已经把 RF 有效送入 AD8310。

### 10.2 Pluto 经 TPin 输入

接法：

```text
Pluto TX1A 同轴中心 -> 1k ohm -> AD8310 TPin
Pluto TX1A 同轴外壳 -> AD8310 TPcom
右 SMA 不接
左 SMA 不接
```

400 MHz LO + 1 MHz tone，即 RF 约 401 MHz：

```text
TPVout 全程约 2.73 V，无变化
```

100 MHz LO + 1 MHz tone，即 RF 约 101 MHz：

```text
TX OFF:
TPVout = 2.72 V

TX ON -10dB:
TPVout = 2.72 V
```

结论：

- Pluto RF 在 401 MHz 和 101 MHz 条件下，即使绕过左 SMA、经 1k 到 TPin，也没有造成 TPVout 可观测变化。
- 可能原因包括：1k 串联电阻导致 RF 过度衰减、TPin 不是合适 RF 注入点、模块输入端需要特定端接/耦合/跳线、AD8310 工作区或 Btrim 配置不合适。

## 11. 当前可靠结论与未证实结论

已经可靠确认：

1. Pluto TX1A 正常，TX1A -> RX1A 回环峰值明显，Peak - Median = 57.5 dB。
2. 同轴线正常，至少在 Pluto 回环中有效。
3. AD8310 模块供电正常，VCC 约 5.0 V。
4. AD8310 空载直流偏置稳定：TPin 约 1.7 V，TPVout 约 2.7 V。
5. 右 SMA 是 TPVout 输出端，禁止作为 RF 输入端。
6. 左 SMA 是输入候选端，与 TPin 有约 2.5k ohm 关系。
7. TPin 直流偏置会影响 TPVout，因此 GPIO40/3.3V 注入结果不能直接解释为 RF 检波。
8. 当前所有 Pluto RF 输入测试下，TPVout 基本卡在约 2.72~2.75 V。

仍未证实：

1. 左 SMA 是否确实是该模块当前配置下的有效 RF 输入端。
2. TPin 是否适合直接注入 RF。
3. AD8310 Figure 38 DC-Coupled Log Amp v2 模块是否需要 50 ohm 端接、隔直电容、跳线、负载或特定 Btrim 设置。
4. Pluto 当前输出功率是否足以进入 AD8310 可检测范围。
5. AD8310 芯片本体是否损坏，或模块是否存在装配/焊接/接口定义问题。

当前问题已经收敛为：

```text
为什么 AD8310 Figure 38 DC-Coupled Log Amp v2 模块在 Pluto RF 输入下 TPVout 不变化？
```

## 12. 下一步最小变量测试

下一步不要再碰右 SMA，不要再盲目加功率。

先确认诊断记录中的串联电阻差异：

```text
确认 GPIO40 / Pluto 接入 TPin 或左 SMA 时，实际使用的是 1k 还是 10k。
```

推荐测试 1：降低 TPin 串联电阻，只改一个变量。

```text

Pluto TX1A 同轴外壳 -> AD8310 TPcom

```

Python 参数保持：

```text

```

```text

TPVout =

TPVout =

TX ON -10dB:

```

判断：

```text

    之前 1k / 10k 串联电阻把 RF 衰减太多。

如果 100 ohm 仍无变化：
    Pluto RF 进入 TPin 也无法触发检波，问题更可能在模块 RF 工作方式、输入耦合、板级配置或接口定义。
```

推荐测试 2：确认输出端不要被负载拉死。

```text

```

推荐测试 3：检查 Btrim 对 TPVout 的影响。

```text
无 RF 输入时，缓慢调 Btrim。
记录 TPVout 是否能随 Btrim 大范围变化。
如果 TPVout 基本不变，说明 Btrim 可能不是当前问题关键。
如果 TPVout 可调，后续应把空载 TPVout 调到参考资料推荐工作点后再测 RF。
```
