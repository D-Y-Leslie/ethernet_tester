## 3. PoE 模块在任务书里的作用

PoE 是 Power over Ethernet，即通过网线同时传输数据和电源。普通网线测试仪只看线序、通断、短路、长度等；加入 PoE 检测后，仪器还能判断这根线或这个端口上是否存在以太网供电电压。

对本科毕设来说，PoE 模块的作用是让测试仪更接近真实工程场景：

```text
1. 判断网口或线路是否带 PoE 电压。
2. 显示 PoE 电压大概是多少。
3. 判断供电可能走 Mode A 还是 Mode B。
4. 避免用户把带电 PoE 线路直接接入普通 GPIO 测试口。
```

小白理解：它像一个“网线电压探测功能”。有些摄像头、AP、门禁设备只插一根网线就能供电，PoE 模块就是用来判断这根网线里有没有这类供电，以及供电大概走哪几根线。

## 4. 为什么会有两个检测通道

PoE 常见供电有两种线组：

```text
Mode A:
  检测 RJ45 1/2 与 3/6 之间的电压

Mode B:
  检测 RJ45 4/5 与 7/8 之间的电压
```

所以完整显示 `MODE_A / MODE_B / MODE_A+B` 时，需要两个相同的检测通道：

```text
通道 A -> 测 VA -> GPIO2
通道 B -> 测 VB -> GPIO13
```

这不是两个复杂模块，而是同一个小电路复制两份。第二份只是把输入脚和 ADC 脚换掉。

## 5. 一个通道是否足够

如果只问本科毕设工作量，一个 Mode A 通道可以支撑“PoE 电压检测原理验证”：

```text
能证明：
  高压经二极管整流和高阻分压后可以安全送入 ESP32 ADC
  ESP32 可以换算并显示 PoE 候选电压
  TJC 页面可以显示 PoE 检测结果

不能证明：
  自动区分 Mode A 和 Mode B
  检测 A+B 双模式
  完整覆盖所有 PoE 供电线组
```

结论：如果时间紧，一个通道加清晰说明已经能作为原理验证；如果要让答辩表述更完整，建议再复制一份 Mode B 通道。因为 Mode B 的工作量很低，主要是重复焊接和重复测试。

## 6. 当前硬件方案

当前采用“两路差分整流 + 高阻分压”的结构。因为暂时只有一块已焊好并验证的电路，所以当前硬件状态是：

```text
Mode A 已搭建并验证
Mode B 尚未搭建
```

当前实际使用的电阻：

```text
R_TOP = 1M
R_BOTTOM = 51K
GPIO 串联保护电阻 = 47K
滤波电容 = 100nF，如无可先不接
```

估算：

```text
5 V USB 测试输入 -> ADC 点约 0.18 V 到 0.22 V
48 V PoE 输入 -> ADC 点约 2.2 V 到 2.4 V
57 V PoE 输入 -> ADC 点约 2.6 V 到 2.8 V
```

由于使用二极管整流，低压测试时会有二极管压降，5 V 输入不会严格按理想分压计算。

## 7. Mode A 当前接线

Mode A 输入：

```text
RJ45 pin1 -> 二极管桥 AC1
RJ45 pin3 -> 二极管桥 AC2
```

用 4 个二极管代替整流桥：

```text
D1: pin1 阳极 -> OUT_A+ 阴极
D2: pin3 阳极 -> OUT_A+ 阴极
D3: OUT_A- 阳极 -> pin1 阴极
D4: OUT_A- 阳极 -> pin3 阴极
```

分压与 ADC：

```text
OUT_A+ -> 1M -> ADC_A_RAW -> 51K -> OUT_A-
ADC_A_RAW -> 47K -> GPIO2
OUT_A- -> ESP32 GND
GPIO2 -> 100nF -> GND   （可选）
```

注意：`OUT_A-` 必须接 ESP32 GND，否则 ADC 没有共同参考点。

## 8. 当前实测结果

断电电阻检查：

```text
ADC_A_RAW 到 OUT_A-/GND = 51.04K
OUT_A+ 到 ADC_A_RAW ≈ 0.995M
OUT_A+ 到 OUT_A-/GND ≈ 1.05M
```

USB 5 V 输入测试：

```text
V(pin1, pin3) ≈ 4.7 V
V(OUT_A+, OUT_A-) ≈ 3.9 V
V(ADC_A_RAW, OUT_A-) ≈ 0.19 V
GPIO2 对 GND ≈ 0.192 V
```

判断：

```text
Mode A 二极管方向正常
分压比例正常
GPIO2 未明显拉低
ESP32 接入安全
```

之前出现 ESP32 发烫，根因是连接错误导致大电流路径；修正后未再出现该问题。

## 9. 下一步硬件工作

优先顺序：

```text
1. 保持 Mode A 不再反复改线，先进入软件诊断草图。
2. 用 GPIO2 读取当前 0.19 V，确认 ADC 读数与万用表一致。
3. 若时间允许，复制 Mode A 电路做 Mode B。
4. Mode B 使用 RJ45 pin4 / pin7，ADC 接 GPIO13。
```

Mode B 复制关系：

```text
RJ45 pin4 -> 二极管桥 AC1
RJ45 pin7 -> 二极管桥 AC2
OUT_B+ -> 1M -> ADC_B_RAW -> 51K -> OUT_B-
ADC_B_RAW -> 47K -> GPIO13
OUT_B- -> ESP32 GND
```

如果暂时不做 Mode B，软件页面中 `VB` 可先显示 `-- V` 或 `N/A`，但论文和答辩中要写清楚“已完成 Mode A 候选供电对检测验证”。
